## Первый запуск

### 1. Установка зависимостей

```powershell
pip install -r requirements.txt
```

### 2. Создание файла .env

```powershell
Copy-Item .env.example .env
```

### 3. Инициализация базы данных

```powershell
python -c "from db import Database; db = Database(); db.init_schema(); print('БД готова'); db.close()"
```

### 4. Запуск приложения

```powershell
python main.py
```

## Настройка API-ключей

ChatList поддерживает следующие AI модели:

### OpenAI (GPT-4, GPT-3.5)

1. Зарегистрируйтесь на https://platform.openai.com
2. Создайте API-ключ: https://platform.openai.com/api-keys
3. Добавьте в `.env`:
```env
OPENAI_API_KEY=sk-proj-ваш_ключ_здесь
```

### DeepSeek

1. Зарегистрируйтесь на https://platform.deepseek.com
2. Создайте API-ключ: https://platform.deepseek.com/api-keys
3. Добавьте в `.env`:
```env
DEEPSEEK_API_KEY=ваш_ключ_здесь
```

### Groq (Llama, Mixtral)

1. Зарегистрируйтесь на https://console.groq.com
2. Создайте API-ключ: https://console.groq.com/keys
3. Добавьте в `.env`:
```env
GROQ_API_KEY=ваш_ключ_здесь
```

### Anthropic (Claude)

1. Зарегистрируйтесь на https://console.anthropic.com
2. Создайте API-ключ: https://console.anthropic.com/settings/keys
3. Добавьте в `.env`:
```env
ANTHROPIC_API_KEY=ваш_ключ_здесь
```

## Основные функции

### Отправка промпта

1. Введите ваш вопрос или запрос в текстовое поле
2. Опционально: добавьте теги для категоризации (через запятую)
3. Нажмите кнопку **"Отправить"** или Ctrl+Enter
4. Дождитесь получения ответов от всех активных моделей
5. Результаты отобразятся в таблице

### Просмотр результатов

Таблица результатов содержит:
- **Чекбокс** - выбор результата для сохранения
- **Модель** - название AI модели
- **Ответ** - текст ответа (наведите курсор для полного текста)
- **Время** - время получения ответа в секундах
- **Токены** - количество использованных токенов

### Сохранение результатов

1. Отметьте чекбоксы нужных результатов
2. Нажмите **"Сохранить выбранные"**
3. Результаты сохранятся в базу данных

**По умолчанию:** все успешные ответы отмечены для сохранения.

### Очистка

- **"Очистить"** (вверху) - очистить поле ввода промпта
- **"Очистить результаты"** (внизу) - очистить таблицу результатов

## Управление моделями

**Меню → Управление → Управление моделями**

### Активация/деактивация моделей

1. Откройте окно управления моделями
2. Установите/снимите чекбокс в колонке "Активна"
3. Только активные модели будут получать запросы

### Проверка статуса

Колонка **"Статус"** показывает:
- ✅ **Настроен** - API-ключ найден в .env
- ⚠️ **Ключ не найден** - нужно добавить ключ в .env

### Добавление своей модели

1. Нажмите **"Добавить модель"**
2. Заполните поля:
   - **Название** - любое удобное имя
   - **API URL** - эндпоинт API
   - **Переменная ключа** - имя переменной в .env
   - **ID модели** - идентификатор модели в API
3. Нажмите **OK**

**Пример:**
```
Название: My Custom Model
API URL: https://api.example.com/v1/chat/completions
Переменная ключа: MY_API_KEY
ID модели: custom-model-v1
```

Затем добавьте в `.env`:
```env
MY_API_KEY=ваш_ключ
```

### Удаление модели

1. Выберите модель в таблице
2. Нажмите **"Удалить"**
3. Подтвердите удаление

⚠️ **Внимание:** это также удалит все сохранённые результаты от этой модели.

## История промптов

**Кнопка "История"** или **Меню → Управление → История промптов**

### Просмотр истории

- Все отправленные промпты сохраняются автоматически
- Таблица показывает дату, текст промпта и теги

### Поиск

Введите текст в поле поиска для фильтрации промптов по содержимому или тегам.

### Использование сохранённого промпта

1. Выберите промпт в таблице
2. Нажмите **"Выбрать"** или дважды кликните
3. Промпт вставится в поле ввода

### Удаление промптов

1. Выберите промпт
2. Нажмите **"Удалить"**
3. Подтвердите удаление

⚠️ **Внимание:** это также удалит все связанные результаты.

## Экспорт результатов

**Кнопка "Экспорт..."** или **Меню → Файл → Экспорт результатов**

### Форматы экспорта

#### Markdown (.md)
```markdown
# Результаты ChatList

## 1. GPT-4 Turbo

**Время ответа:** 2.34с
**Токенов:** 150

**Ответ:**
[текст ответа]
```

#### JSON (.json)
```json
{
  "export_date": "2026-01-02T12:00:00",
  "results": [
    {
      "model_name": "GPT-4 Turbo",
      "response_text": "...",
      "response_time": 2.34,
      "tokens_used": 150
    }
  ]
}
```

#### Текст (.txt)
Простой текстовый формат для удобного чтения.

### Процесс экспорта

1. Нажмите **"Экспорт..."**
2. Выберите формат
3. Укажите место сохранения
4. Нажмите **"Сохранить"**

## Настройки

**Меню → Управление → Настройки**

### Доступные настройки

#### Таймаут запросов (сек)
Максимальное время ожидания ответа от API.
- **По умолчанию:** 30 секунд
- **Диапазон:** 5-120 секунд

#### Максимум повторных попыток
Количество повторных попыток при ошибке сети.
- **По умолчанию:** 3 попытки
- **Диапазон:** 0-10 попыток

#### Тема
Оформление интерфейса.
- **light** - светлая тема
- **dark** - тёмная тема (в разработке)

#### Формат экспорта по умолчанию
Формат, который будет предложен при экспорте.
- markdown
- json
- txt

### Применение настроек

После сохранения настроек **перезапустите приложение**.

## Решение проблем

### Ошибка: "Нет активных моделей"

**Решение:**
1. Откройте **Управление → Управление моделями**
2. Активируйте нужные модели (чекбокс)

### Ошибка: "Не найдены API-ключи"

**Решение:**
1. Убедитесь, что файл `.env` существует
2. Проверьте правильность написания имён переменных
3. Убедитесь, что значения не содержат пробелов
4. Перезапустите приложение

**Пример правильного .env:**
```env
OPENAI_API_KEY=sk-proj-abc123...
DEEPSEEK_API_KEY=sk-abc123...
```

### Ошибка: "DLL load failed" (Windows)

Это ошибка PyQt6 на Windows. Мы используем PyQt5 вместо PyQt6.

**Решение:**
```powershell
pip uninstall PyQt6 -y
pip install PyQt5==5.15.10
```

### Медленная работа / долгое ожидание

**Причины:**
- Медленный интернет
- API перегружен
- Большой промпт

**Решение:**
1. Увеличьте таймаут в настройках
2. Уменьшите количество активных моделей
3. Сократите текст промпта

### База данных повреждена

**Решение:**
```powershell
# Создать резервную копию
Copy-Item chatlist.db chatlist_backup.db

# Пересоздать БД
Remove-Item chatlist.db
python -c "from db import Database; db = Database(); db.init_schema(); db.close()"
```

### Приложение не запускается

**Проверьте:**
1. Установлен Python 3.11+
2. Установлены все зависимости: `pip install -r requirements.txt`
3. Нет конфликтов версий: `pip list`

**Переустановка:**
```powershell
pip install --force-reinstall -r requirements.txt
```

## Советы по использованию

### 1. Эффективное использование тегов

Добавляйте теги для категоризации промптов:
```
python, debugging
marketing, email
translation, english
```

Это упростит поиск в истории.

### 2. Сравнение моделей

Отправляйте один промпт в несколько моделей для:
- Сравнения качества ответов
- Проверки фактов
- Выбора лучшего варианта

### 3. Сохранение лучших результатов

Не сохраняйте все ответы - выбирайте только лучшие. Это сэкономит место в БД.

### 4. Регулярный экспорт

Периодически экспортируйте важные результаты в Markdown или JSON для резервного копирования.

### 5. Тестирование новых моделей

Добавляйте новые модели и сравнивайте их с существующими на одинаковых промптах.
